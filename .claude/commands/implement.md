## 実装フェーズ

あなたは**シニアフルスタックエンジニア**として、仕様に基づいた実装を行ってください。

### 0. 実装開始前の確認

#### 必須チェック
- [ ] `.tmp/requirements.md` が完成している
- [ ] `.tmp/design.md` が完成している
- [ ] `.tmp/tasks.md` にPhase 1タスクが定義されている
- [ ] ユーザーが仕様を承認済み

**重要**: 仕様が未完成の場合、先に `/spec` を実行してください。

### 1. 実装ルール

#### コーディング規約
- TypeScript strict mode 必須
- `any`型の使用禁止
- 関数型アプローチ優先（副作用を局所化）
- コンポーネントはServer Componentsをデフォルトに
- `'use client'`は必要な場合のみ

#### 禁止事項
- [ ] 仕様にない機能の追加（スコープクリープ防止）
- [ ] テストなしでのビジネスロジック実装
- [ ] コメントなしの複雑なアルゴリズム
- [ ] ハードコーディングされた設定値

#### 推奨事項
- 小さなコミット単位（1機能 = 1コミット）
- 型定義ファーストでの実装
- エラーハンドリングの徹底

### 2. 実装の進め方

#### タスク管理
1. `.tmp/tasks.md` の未完了タスクを確認
2. 優先度・依存関係に従って1つずつ実装
3. 完了したら即座にチェック（`- [x]`）
4. ブロッカーがあれば記録

#### 実装サイクル
```
1. タスク選択
2. 型定義（types/）
3. ロジック実装（lib/）
4. テスト作成・実行
5. UIコンポーネント実装（components/）
6. 統合・動作確認
7. コミット
8. 次のタスクへ
```

### 3. 品質チェック

#### 各タスク完了時
```bash
# 型チェック
npm run type-check

# Lint
npm run lint

# テスト
npm run test
```

#### Phase完了時
```bash
# ビルド確認
npm run build

# 開発サーバーでの動作確認
npm run dev
```

### 4. コミット規約

```bash
git commit -m "feat: {{FEATURE_NAME}}

- {{変更点1}}
- {{変更点2}}

Phase: 1
Task: FR-1.x

🤖 Generated with Claude Code"
```

### 5. Phase完了時の報告

#### 報告内容
1. 完了したタスク一覧
2. 実装した機能のサマリ
3. 発生した問題と解決策
4. 次のPhaseへの引き継ぎ事項

#### ユーザー確認ゲート
- [ ] 実装した機能のデモ/確認
- [ ] 次のPhaseへ進む承認

---

**注意**: 各Phase完了時はユーザー確認を必ず挟むこと。
