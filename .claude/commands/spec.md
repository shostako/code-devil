## 仕様策定フェーズ

あなたは**要件アナリスト兼アーキテクト**として、プロジェクトの仕様を詳細化してください。

### 1. 要件定義の完成

#### `.tmp/requirements.md` の更新
以下のセクションを具体的な内容で埋めてください：

1. **プロジェクト概要**
   - 目的を1-2文で明確化
   - 解決する課題を3-5個列挙
   - コアバリューを優先順位付け

2. **ユーザーストーリー（EARS形式）**
   - 主要な操作ごとにストーリーを作成
   - When/the system shall/so that 形式で記述
   - 受け入れ基準を具体的に定義

3. **機能要件**
   - Phase 1（MVP）の必須機能を定義
   - 各機能にID（FR-1.1等）を付与
   - 優先度と詳細を記載

4. **非機能要件**
   - 性能目標（応答速度、ロード時間）
   - 互換性（ブラウザ、デバイス）
   - アクセシビリティ基準

5. **成功基準**
   - Phase別の達成条件をチェックリスト化

### 2. 技術設計の完成

#### `.tmp/design.md` の更新
以下のセクションを具体的な内容で埋めてください：

1. **技術スタック選択**
   - 各技術の選定理由を明記
   - 不採用技術とその理由も記録
   - 将来の拡張可能性を考慮

2. **システムアーキテクチャ**
   - 全体構成図をASCIIアートで作成
   - 各レイヤーの責務を明確化
   - データフローを図示

3. **データモデル**
   - TypeScriptインターフェースで定義
   - 各フィールドの用途をコメント

4. **テスト戦略**
   - テスト種別と担当範囲
   - カバレッジ目標

### 3. タスクリストの作成

#### `.tmp/tasks.md` の更新
設計に基づいてタスクを具体化：

1. **Phase 1タスクの詳細化**
   - 各機能を実装可能な粒度に分解
   - 依存関係を考慮した順序付け
   - 見積もり時間の記載（任意）

2. **マイルストーン設定**
   - 目標完了日を設定
   - ブロッカーリスクを列挙

### 4. ユーザー確認ゲート

#### 確認事項
仕様策定完了後、ユーザーに以下を確認してください：

- [ ] 要件定義の内容に問題はないか
- [ ] 技術選択に異議はないか
- [ ] タスクの粒度は適切か
- [ ] スケジュールは現実的か

**重要**: ユーザーの承認なしに実装フェーズに進まないこと。

### 5. 仕様策定完了の出力

以下をユーザーに報告してください：
1. 要件定義のサマリ（主要機能3-5個）
2. 技術スタックの概要
3. Phase 1のタスク数と見積もり
4. 確認を求める質問リスト

---

**次のステップ**: ユーザー承認後、`/implement` コマンドで実装を開始
