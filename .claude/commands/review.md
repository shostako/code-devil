## レビュー・検証フェーズ

あなたは**品質保証エンジニア**として、実装の品質を検証してください。

### 1. 機能検証

#### 要件充足確認
`.tmp/requirements.md` の成功基準を1つずつ確認：

- [ ] Phase 1の成功基準をすべて満たしている
- [ ] ユーザーストーリーの受け入れ基準を満たしている
- [ ] 非機能要件（性能・互換性）を満たしている

#### 動作確認
各機能について以下を確認：

| 機能 | 正常系 | 異常系 | エッジケース |
|------|--------|--------|-------------|
| FR-1.1 | ✅/❌ | ✅/❌ | ✅/❌ |
| FR-1.2 | ✅/❌ | ✅/❌ | ✅/❌ |
| ... | ... | ... | ... |

### 2. コード品質検証

#### 静的解析
```bash
# 型チェック
npm run type-check

# Lint
npm run lint

# テスト + カバレッジ
npm run test:coverage
```

#### 品質基準
- [ ] TypeScript strict mode でエラーなし
- [ ] ESLint警告なし（または許容範囲内）
- [ ] テストカバレッジ目標達成
- [ ] `any`型の使用がない

#### コードレビュー観点
- [ ] 命名規則の一貫性
- [ ] 関数の単一責任
- [ ] エラーハンドリングの適切さ
- [ ] コメントの適切さ（過不足なし）
- [ ] 重複コードの排除

### 3. パフォーマンス検証

#### 計測項目
- [ ] 初回ロード時間: __ms（目標: {{TARGET}}ms以内）
- [ ] 主要操作の応答時間: __ms（目標: {{TARGET}}ms以内）
- [ ] バンドルサイズ: __KB

#### ツール
```bash
# ビルドサイズ確認
npm run build
# → .next/analyze または webpack-bundle-analyzer

# Lighthouse（ブラウザで実行）
# Performance, Accessibility, Best Practices, SEO
```

### 4. セキュリティ検証

#### チェックリスト
- [ ] XSS対策（ユーザー入力のサニタイズ）
- [ ] 機密情報がコードに含まれていない
- [ ] 依存パッケージに既知の脆弱性がない（`npm audit`）

```bash
npm audit
```

### 5. ドキュメント検証

#### 確認項目
- [ ] README.mdが最新の状態
- [ ] CLAUDE.mdがプロジェクトを正確に反映
- [ ] `.tmp/`の仕様書が実装と一致
- [ ] 作業ログ（logs/）が記録されている

### 6. 検証結果報告

#### 報告フォーマット
```markdown
## 検証結果サマリ

### 合格項目
- ✅ 機能要件充足
- ✅ コード品質基準達成
- ✅ パフォーマンス目標達成

### 要改善項目
- ⚠️ {{ISSUE_1}}: {{DETAIL}}
- ⚠️ {{ISSUE_2}}: {{DETAIL}}

### 推奨アクション
1. {{ACTION_1}}
2. {{ACTION_2}}
```

### 7. リリース判定

#### リリース可能条件
- [ ] すべての機能要件を満たしている
- [ ] 重大なバグがない
- [ ] パフォーマンスが目標を達成
- [ ] ドキュメントが完備

#### 判定
- **GO**: リリース可能
- **CONDITIONAL GO**: 軽微な修正後リリース可能
- **NO GO**: 重大な問題あり、修正必要

---

**完了後**: 検証結果をユーザーに報告し、リリースまたは修正の判断を仰ぐ
