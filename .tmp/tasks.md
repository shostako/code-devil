## Claude Code最適化制約

### タスク粒度ルール
- **1タスク = 1-3時間**（コンテキスト内完結）
- **1タスク = 3-5ファイル以下**の変更
- **依存関係**を明確にし、並行実行可能なタスクを識別

### 並行実行の活用
```
Task A（ロジック実装）←→ Task B（UI実装）
  └─ インターフェース合意済みなら並行可能
```

## タスク分解品質チェックリスト

### 1. 粒度
- [ ] 全タスクが1-3時間で完了可能
- [ ] 1タスクあたりの変更ファイル数が3-5以下
- [ ] 各タスクの完了条件が明確
- [ ] 依存関係が明示済み

### 2. 網羅性
- [ ] 要件定義の全機能がカバーされている
- [ ] 設計書の全コンポーネントが実装対象
- [ ] テストタスクが含まれている
- [ ] ドキュメント更新タスクが含まれている

### 3. 実行可能性
- [ ] 各タスクが独立して検証可能
- [ ] 並行実行可能なタスクが識別済み
- [ ] ブロッカーとなる依存関係が特定済み
- [ ] リスクの高いタスクが早期にスケジュール済み

### 4. 品質確認
- [ ] 各Phaseに品質チェックポイントがある
- [ ] lint/typecheck実行タスクが含まれている
- [ ] ユーザー確認ポイントが設定済み
- [ ] 最終検証タスクが定義済み
