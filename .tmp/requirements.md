## 厳格な品質チェックリスト

要件定義完了前に**全項目**を確認してください：

### 1. 実装要求仕様
- [ ] 各機能に「完了条件: [測定可能な基準]」が記載済み
- [ ] 全機能の入力・処理・出力が具体的に定義済み
- [ ] 必須機能とオプション機能の線引きが明確
- [ ] 全機能に実装優先度(高/中/低)が設定済み

### 2. UI/UX具体化
- [ ] Primary FlowとAlternative Flowsが3ステップ以上で記載
- [ ] 各UI要件に測定可能な完了条件が設定済み
- [ ] 「3クリック以内」等の具体的な効率目標設定済み
- [ ] 想定エラーとその表示方法が明記済み

### 3. Claude Code最適化
- [ ] 使用予定のClaude Codeツールが明記済み
- [ ] lint/typecheck等の品質確認方法が定義済み
- [ ] プロジェクト構造の概要が想定済み
- [ ] 1-3時間単位のタスクに分解可能な詳細度

### 4. 測定可能性
- [ ] 「動作する」でなく「〇〇の入力で△△が表示される」レベル
- [ ] 具体的な受け入れテストシナリオが2つ以上
- [ ] 可能な限り定量的な成功基準を設定
- [ ] 各リスクに具体的な対策方法を記載

### 5. スコープ管理
- [ ] 「含むもの/含まないもの」が明確に分離
- [ ] 技術制約・時間制約・学習コスト制約を現実的に設定
- [ ] 将来拡張の方向性を考慮したアーキテクチャ想定
- [ ] MVPに集中するためのスコープ制限策

### 6. ユーザー確認必須
- [ ] ユーザーが全要件を理解し合意済み
- [ ] 「後で決める」「適宜」等の曖昧表現が0件
- [ ] 技術的実現可能性をユーザーと確認済み
- [ ] 機能の優先順位についてユーザー合意済み
